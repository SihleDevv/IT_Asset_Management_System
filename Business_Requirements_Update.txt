2. Background

2.1 Introduction
The IT Asset Management System (ITAMS) is a .NET (C#) MVC web application designed to capture, manage, and report on IT assets such as computers, servers, software applications, and users within an organization's IT infrastructure. The system stores asset details, records status, tracks assignments, manages user access, and provides comprehensive reporting for management purposes.

2.2 Evolution & Context
Previously, IT assets were tracked using spreadsheets and manual reporting. ITAMS introduces a centralized, role-based system for recording asset information, managing user access, tracking asset assignments, and generating reports efficiently with enhanced security features.

2.4 Statement of Business Requirement
The system must allow authorized users to:
• Register and track computers, servers, software applications, and users
• Record hardware and software properties and asset ownership
• Track the status of assets and their assignments to users
• Manage user accounts with role-based permissions
• Generate reports and dashboards for IT management
• Ensure role-based access for security
• Manage passwords with expiration policies
• View and update personal profiles
• Track audit logs of all system activities
• Create and manage IT support tickets for asset issues
• Assign and track support tickets through resolution workflow
• Request asset replacements when assets are beyond repair
• Communicate follow-up information on support tickets

7. Implemented Features

7.1 Asset Management
• Create, Read, Update, and Delete operations for three asset types:
  - Computers (with specifications: Processor, RAM, Storage, Operating System)
  - Servers (with server type, operating system, and project manager assignment)
  - Software Applications (with category, business unit, version, and license information)
• Dynamic vendor dropdown based on asset type selection
• Asset assignment to users (computers assigned to users, servers managed by project managers, applications owned by application owners)
• Asset status tracking (Available, In Use, Under Maintenance, Retired, Damaged for computers; Online, Offline, Maintenance, Decommissioned for servers; Enabled, Disabled, Expired, Deprecated for applications)
• Asset dependency management (prevents deletion of assets with active dependencies)
• Asset filtering by user role (regular users see only their assigned assets; Admin/IT Manager see all assets)
• Search and filter functionality for assets
• CSV export functionality for asset data

7.2 User Management
• User registration with role assignment (Admin, IT Manager, Employee, Read Only)
• User profile management (view, edit profile information)
• User list with search and filter capabilities
• User deletion with dependency checks
• Prevention of self-deletion for Admin/IT Manager users
• User status management (Active/Inactive)
• CSV export for user data

7.3 Role-Based Access Control
• Six user roles with distinct permissions:
  - Admin: Full system access including user creation, asset management, password reset, and deletion rights
  - IT Manager: Can view and edit assets, manage users (view only), but cannot create users or delete assets
  - IT Support Supervisor: Can assign tickets to technicians, view all tickets, manage ticket assignments, and respond to replacement requests
  - IT Support (Technician): Can action assigned tickets with status transitions (Pending → In Progress → Resolved), add technician notes, request asset replacements, but cannot skip status transitions
  - Employee: Can view assigned assets, create support tickets, add follow-up notes, and update own profile
  - Read Only: Can view assigned assets only
• Role-based menu visibility (Reports visible only to Admin/IT Manager; Logs visible only to Admin/IT Manager; Assets menu hidden for IT Support roles)
• Role-based asset visibility (regular users see only assigned assets; Admin/IT Manager see all assets)
• Role-based action buttons (Delete buttons visible only to Admin)
• Role-based ticket access (users see their own tickets; IT Support see assigned/unassigned tickets; supervisors see all tickets)

7.4 Password Management
• Password policy enforcement (8-30 characters, at least one digit, one special character from [~!@#$^&*?><])
• Password expiration after 30 days
• Automatic password expiration check on login
• Forced password change for expired passwords
• Admin password reset functionality for all users
• Password management page for Admin (view all users, search, filter, reset passwords)
• Password status tracking (Valid, Expiring Soon, Expired, Never Changed)
• Password change history tracking

7.5 Profile Management
• View-only profile page displaying user information
• Edit profile functionality (separate from password change)
• Change password functionality (requires current password verification)
• Role and department editing (Admin and IT Manager can edit their own role and department; Read Only and Employee cannot)
• Profile update logging in audit trail

7.6 Dashboard
• Asset summary statistics (Total Computers, Servers, Applications)
• Server status overview (Active/Inactive servers)
• Computer status overview (In Use/Available computers)
• License expiration warnings (Expiring Soon/Expired licenses)
• Ticket metrics for IT Support roles:
  - IT Support Supervisor: Total number of all tickets
  - IT Support Technician: Number of tickets assigned to them
• Recent activities log (visible to Admin and IT Manager only)
• Role-based dashboard content visibility
• Role-based metrics filtering (regular users see only assigned assets; Admin/IT Manager see all assets)

7.7 Reporting
• Asset Summary Report
• Computer Report
• Server Report
• Application Report
• License Report
• Audit Log Report (Admin and IT Manager only)
• Reports accessible only to Admin and IT Manager roles

7.8 Server Application Management
• Install applications on servers
• Uninstall applications from servers
• Update application status on servers (Running, Stopped, Maintenance, Error)
• View installed applications on servers
• Application installation date and version tracking
• Notes and status management for installed applications

7.9 Audit Logging
• Comprehensive audit trail of all system activities
• Logged actions include:
  - User login/logout
  - Asset creation, update, deletion
  - User creation, update, deletion
  - Password changes and resets
  - Profile updates
  - Application installations/uninstallations
  - Server application status updates
• Audit log includes: User name, action, entity type, entity ID, details, IP address, timestamp
• Audit log access restricted to Admin and IT Manager

7.10 Asset Dependencies
• Computer-User dependency (prevents deletion of computers assigned to users, except "Unassigned")
• Server-Application dependency (prevents deletion of servers with installed applications)
• Application-Server dependency (prevents deletion of applications installed on servers)
• User-Computer dependency (prevents deletion of users with assigned computers)
• Clear dependency messages with guidance for resolution

7.11 IT Support Ticket System
• Ticket creation by users for their assigned assets (Computer, Server, or Application)
• Required asset selection during ticket creation
• Automatic assignment of new tickets to IT Support Supervisor
• Ticket assignment workflow:
  - IT Support Supervisor can assign tickets to IT Support Technicians
  - IT Support Technicians can only action tickets assigned to them
• Status workflow enforcement:
  - Status transitions: Pending → In Progress → Resolved → Closed
  - IT Support Technicians cannot skip from Pending to Resolved (must go through In Progress)
  - Resolution Notes required when resolving tickets
• Ticket priority levels: Low, Medium, High, Critical
• Ticket expiration warnings:
  - WARNING badge for tickets 2-3 days old (Pending/In Progress status)
  - EXPIRED badge for tickets over 3 days old (Pending/In Progress status)
• Technician Notes with timestamp tracking
• Status change tracking with timestamp and user who changed status
• Last Action Date tracking for all ticket modifications
• Replacement Request System:
  - Technicians can request asset replacement when asset is beyond repair
  - Admin/IT Manager can approve or respond to replacement requests
  - Replacement reason and admin response tracking
• User Follow-Up System:
  - Users can add follow-up notes to their tickets
  - Follow-up date tracking
• Ticket filtering and search:
  - Filter by status (All, Pending, In Progress, Resolved, Closed)
  - Filter by priority (All, Low, Medium, High, Critical)
  - Search functionality
• CSV export of filtered tickets (all user roles)
• "Assigned To" column display for all roles
• "Report Issue" button on asset detail pages for quick ticket creation
• Ticket details page with full ticket information and history
• Role-based ticket visibility:
  - Users see only their own tickets
  - IT Support Technicians see tickets assigned to them and unassigned tickets
  - IT Support Supervisors see all tickets
  - Admin/IT Manager see all tickets

8. Functional Requirements

8.1 Authentication & Authorization
FR-001: The system shall require user authentication via email and password before accessing any functionality.
FR-002: The system shall support six user roles: Admin, IT Manager, IT Support Supervisor, IT Support (Technician), Employee, and Read Only.
FR-003: The system shall enforce role-based access control for all system features.
FR-004: The system shall prevent unauthorized access to restricted features through direct URL access.
FR-005: The system shall maintain user sessions with configurable timeout (8 hours with sliding expiration).

8.2 User Role Management
FR-005A: The system shall allow Admin to create users with IT Support Supervisor and IT Support (Technician) roles.
FR-005B: The system shall restrict IT Support role creation to Admin only.

8.3 Password Management
FR-006: The system shall enforce password policy: 8-30 characters, at least one digit, one special character [~!@#$^&*?><].
FR-007: The system shall track password change dates for all users.
FR-008: The system shall automatically expire passwords after 30 days.
FR-009: The system shall force users to change expired passwords on login.
FR-010: The system shall allow Admin to reset passwords for any user.
FR-011: The system shall display password status (Valid, Expiring Soon, Expired, Never Changed) in password management.
FR-012: The system shall require current password verification when users change their own password.

8.4 User Management
FR-013: The system shall allow Admin to create new users with role assignment.
FR-014: The system shall allow Admin and IT Manager to view user lists.
FR-015: The system shall prevent IT Manager from creating new users.
FR-016: The system shall allow Admin to edit user information.
FR-017: The system shall allow Admin to delete users (with dependency checks).
FR-018: The system shall prevent users from deleting their own accounts.
FR-019: The system shall support user search by email, full name, department, or username.
FR-020: The system shall export user data to CSV format.

8.5 Asset Management - Computers
FR-021: The system shall allow Admin to create, read, update, and delete computer assets.
FR-022: The system shall capture computer specifications: Processor, RAM, Storage, Operating System.
FR-023: The system shall allow assignment of computers to users.
FR-024: The system shall track computer status: Available, In Use, Under Maintenance, Retired, Damaged.
FR-025: The system shall prevent deletion of computers assigned to users (except "Unassigned").
FR-026: The system shall display computers assigned to regular users (Employee, Read Only).
FR-027: The system shall display all computers to Admin and IT Manager.

8.6 Asset Management - Servers
FR-028: The system shall allow Admin to create, read, update, and delete server assets.
FR-029: The system shall capture server details: Server Type, Operating System, Project Manager Name.
FR-030: The system shall allow optional purchase price and warranty expiry date for servers.
FR-031: The system shall track server status: Online, Offline, Maintenance, Decommissioned.
FR-032: The system shall prevent deletion of servers with installed applications.
FR-033: The system shall display servers managed by regular users (as Project Manager).
FR-034: The system shall display all servers to Admin and IT Manager.

8.7 Asset Management - Software Applications
FR-035: The system shall allow Admin to create, read, update, and delete software application assets.
FR-036: The system shall capture application details: Category, Business Unit, Version, Application Owner.
FR-037: The system shall support license information: License Type, License Holder, License Count, Expiry Date.
FR-038: The system shall track application status: Enabled, Disabled, Expired, Deprecated.
FR-039: The system shall prevent deletion of applications installed on servers.
FR-040: The system shall display applications owned by regular users (as Application Owner).
FR-041: The system shall display all applications to Admin and IT Manager.

8.8 Asset Assignment & Visibility
FR-042: The system shall filter asset visibility based on user role.
FR-043: The system shall display only assigned assets to Employee and Read Only users:
  - Computers where AssignedTo matches user's full name or email
  - Servers where ProjectManagerName matches user's full name or email
  - Applications where ApplicationOwner matches user's full name or email
FR-044: The system shall display all assets to Admin and IT Manager.
FR-045: The system shall exclude "Unassigned" from dependency checks.

8.9 Server Application Management
FR-046: The system shall allow Admin and IT Manager to install applications on servers.
FR-047: The system shall allow Admin and IT Manager to uninstall applications from servers.
FR-048: The system shall allow Admin and IT Manager to update application status on servers.
FR-049: The system shall track installation date, version, status, and notes for installed applications.
FR-050: The system shall display installed applications on server details page (read-only for all users).
FR-051: The system shall allow editing of application status and notes on servers.

8.10 Profile Management
FR-052: The system shall provide a view-only profile page for all users.
FR-053: The system shall allow users to edit their profile information (separate from password change).
FR-054: The system shall allow users to change their password (requires current password).
FR-055: The system shall allow Admin and IT Manager to edit their own role and department.
FR-056: The system shall prevent Read Only and Employee users from changing their role or department.
FR-057: The system shall display password last changed date on profile.

8.11 Dashboard & Reporting
FR-058: The system shall display asset summary statistics on the dashboard.
FR-059: The system shall display recent activities log (Admin and IT Manager only).
FR-060: The system shall display ticket metrics on dashboard for IT Support roles:
  - IT Support Supervisor: Total number of all tickets
  - IT Support Technician: Number of tickets assigned to them
FR-061: The system shall filter dashboard metrics by user role (regular users see assigned assets; Admin/IT Manager see all assets).
FR-062: The system shall provide Asset Summary Report.
FR-063: The system shall provide Computer Report.
FR-064: The system shall provide Server Report.

FR-110: The system shall track ticket creation date, updated date, resolved date, and last action date.

10. Security & Access Control

9.1 Authentication
• ASP.NET Core Identity for user authentication
• Email and password-based login
• Session management with 8-hour timeout and sliding expiration
• Secure cookie configuration (HttpOnly, SameSite)

9.2 Authorization
• Role-based access control (RBAC) with six roles:
  - Admin: Full system access
  - IT Manager: View and edit assets, view users (cannot create users or delete assets)
  - IT Support Supervisor: Assign tickets to technicians, view all tickets, manage ticket assignments, respond to replacement requests
  - IT Support (Technician): Action assigned tickets with status transitions, add technician notes, request asset replacements
  - Employee: View assigned assets, create support tickets, add follow-up notes, update own profile
  - Read Only: View assigned assets only
• Policy-based authorization for specific actions
• Action-level authorization attributes
• Role-based menu visibility (Assets menu hidden for IT Support roles)

9.3 Password Security
• Strong password policy enforcement
• Password hashing using ASP.NET Core Identity
• Password expiration after 30 days
• Forced password change for expired passwords
• Admin password reset capability

9.4 Data Protection
• HTTPS encryption for data transmission
• Anti-forgery tokens on all forms
• SQL injection prevention through Entity Framework Core
• Input validation on all user inputs

9.5 Audit & Compliance
• Comprehensive audit logging of all system activities
• IP address tracking for security monitoring
• Timestamp tracking for all actions
• Audit log access restricted to Admin and IT Manager

9.6 Access Restrictions
• Role-based menu visibility
• Role-based asset visibility
• Role-based action button visibility
• Direct URL access prevention through authorization filters
